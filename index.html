Warcraft Player and Guild Spotlight

SITE PURPOSE BACKGROUND

I am a fairly popular World of Warcraft addon developer, as a hobby. My most popular addon is Guild Roster Manager, with close to 75,000 active unique users. This addon collects information on guilds as a whole and allows leadership an opportunity to more-easily manage their guild as it tracks changes to individual players, membership issues, guild notable events, and other important information unique to the game.

One challenge that I have encountered is that the in-game available API is walled-off from the internet. As such, useful information from the game company Blizzard's Web API is not available from within the game, only their own limited toolset. This greatly hinders some more advance things that I want to do with the data collected in-game and saved to a Lua file.

KEY PURPOSE

I want to convert the Lua save variable file into a JSON compatible format, then I want to import that JSON file information my website so that a profile and analysis can be done on the details around the guild and/or the player. Of note, since it is slightly outside the scope of this assignment, the conversion of the desktop save file into a JSON format will be done externally so only a JSON will be imported. But, addon save data format can be VIEWED HERE After importing this data I will be able to do the following:

  • Use an API of a Large Language Model (LLM) to write up a small player analysis on the data I feed it imported from the JSON, adding thematic and Role-playing flare to it, for fun. This can be set to a weekly schedule to update changes since previous week
  • Use the Blizzard Battle.net API to unify inaccessible data from within the game with the battle.net Armory API with missing details (i-Lvl, alts, etc.)
  • STRETCH POSSIBILITY - Do a guild snapshot of all saved full guild data, not just player data with the above features.
  • PAGE 2 - Warcraft Realms - Battle.net Web API demo to access Realm Activity Data
  • PAGE 3 - Details on the Guild Roster Manager addon and how why the addon matters for the site

Practical Usage #1:A fun way to get an AI LLM to do an analysis of your player, in an RPG theme, combined with real in-game data.

Practical Usage #2:Web API integration with in-game accessible data has long been requested by people on the official GRM Discord Server. It is my number one most requested feature. Implementing this will satisfy a lot of people.

Practical Usage #3:Laying the foundation here with being able to import the guild and player data by way of a JSON file opens the door to so many other possibilities, like being able to utilize the data for Discord bots, or other web integrations. Creating my own API that can then be leveraged by guilds who use this addon. For example, the most famous Warcraft e-gaming organization in the world, METHOD, uses this addon and it could be very useful to integrate various in-game addon features to their website, though that is a project for another day. The bones and framework by building just this foundational, proof-of-concept site open the door to more advanced features.

SCENARIOS

Where can I get an LLM to write a fun analaysis on my own toon, or all of my alts?

Where can I get an LLM to write a fun analysis of the Guild as a whole and various updates or accomplishes that occurred that week?

INITIAL MODULE LIST

  • JSON File Parser Module: Imports and parses the JSON file from Guild Roster Manager on the Node.js server (started with node server.js), using fs or jsonfile (via npm).
  • Blizzard Battle.net API Integration Module for player data and realm data(page2): Fetches data (e.g., iLvl, alts) from the Blizzard API via the Node.js server (node server.js), using axios (installed with npm).
  • LLM Module - Maybe OpenAI, or Grok: Sends data to an LLM API for analysis, handled by the Node.js server
  • Frontend Display Module: Renders data on the website, served by node server.js, with code quality maintained via npm format (Prettier) and npm lint (ESLint).
